Git

Hva er git?

Git er et versjonskontroll verktøy som muliggjør det å jobbe med filer lokalt før man commiter og pusher (sender til repo) det til repo. Dette gjør at en utvikler kan gjøre endringer på en fil, commite og pushe til repo og da vil repo lagre den nye versjonen. Hvis utvklikleren nå ønsker å gå tilbake til en eldre versjon er dette mulig uten å endre den nye versjonen. Det er også mulig for mange utvklere å jobbe på samme området og til og med samme filer, det hender at det blir konflikter på systemet når man jobber på samme området. Da vil Git gi beskjed om at det er en konflikt når man pusher filene, da må man pulle (laste ned filene) og merge endringene sammen. Git er vel det ledende versjonskontroll systemet for alle IT prosjekter, dette betyr at hvis man skal jobbe med IT er det en stor fordel å lære seg Git :)

Github

Github er en web hosting service for Git prosjekter, dette betyr at man kan lage repos på github for å slippe å hoste en git server lokalt. Det finnes en gratis versjon og en enterprise versjon av Github, med gratis versjonen er alle prosjektene offentlige. Github er mest brukt til kildekode for prosjekter, men de har også annen funksjonalitet og blir blant annet brukt til issue tracking av noen.

Fordeler med Git

Føler at noen av har blit nevnt allerede, men det er enda flere. Noe av det viktigste er fort at dette er standarden idag, både for de som har små prosjekter og de som har store prosjekter. Mulighetene for å jobbe offline er gode da man kloner repo lokalt og jobber lokalt, det eneste man trenger nettilgang til er for å laste opp endringer og laste ned endringer. Som nevnt tidligere har man flere versjoner, og man har mulighet til å jobbe mot foskjellige versjoner. Man har også mulighet til å jobbe med forskjellige branches i Git, men da begynner konseptet også å bli litt mer komplisert. Hvis datamaskinen din slutter å virke og man må jobbe på en ny maskin så ligger filene/prosjektene dine på repo på en server og alt du trenger å gjøre er å klone repo til lokal maskin.

Hva jeg synes om Git

Så lenge man skjønner tankegangen i Git og hvordan det fungerer så syntes jeg det er kjempebra! Dette avhenger at man må kjenne til pull, push, commit, status, add og clone. Nå har man jo fått grafiske interfaces som er litt bedre å bruke enn git GUI, men jeg syntes kommandolinjen er å foretrekke foreløpig. Anbefaler også andre å prøve kommandolinjen for enkle oppgaver, da det er mulig at man jobber et sted hvor det kun er kommandolinje som gjelder. Funksjonaliteten i Git er også fantastisk, og selv om det kanskje er litt vanskelig i starten er det Git kasnkje det beste verktøyet for prosjektarbeid (og det finnes gratisversjoner som har all funksjonalitet man ønsker).
